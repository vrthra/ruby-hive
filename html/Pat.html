Pat

It is a successor to the Cat DSL. Originally written to test proxy
it can be used for testing CLIs Network Servers and Cients, ProxyServers etc depending on the connectors used.

pat.rb -h
Usage: pat.rb [options]

Specific options:
    -s, --seq=sequence               The sequence of testcases
    -t, --timeout=timeout            The max timeout in seconds
    -p, --proxy=proxyname:port       The proxy against which you want to run the tests
    -r, --serverhost=serverhost:port Server name(opt)
    -v, --verbose=verbose            Run verbosely [1..]
    -d, --[no-]dump                  dump evaluation
    -u, --usedump                    use earlier dumps
    -x, --ext a b c                  use bt-backtrace, delim-dumpdelimmatch|time|xchars

Common options:
    -g, --groups x y z               selected groups
    -h, --help                       Show this message
        --version                    Show version

The -s can be used to specify a sequence of tests.
eg.
|cat all.seq
use 'examples'
use 'fails'


|cat examples.seq
# pat sequence file
[
#drb requires the netagent running in the remote machines
examples/drb [drb]
examples/xml [xml tc]
examples/cli [cli tc]
examples/file [file tc]
examples/httpfetch [fetch tc]
examples/assign [assign tc]
examples/use [use tc]
examples/blocks [block tc]
examples/ftpfetch [tc]
]

[
examples/get [get proxy]
examples/connect [connect proxy]
examples/keepalive [keepalive proxy]
#examples/simplesnmp [snmp]
examples/socks5 [socks5 socks proxy]
examples/socks4 [socks4 socks proxy]
examples/ftpretr [ftp proxy]
]

[
examples/telnet [telnet]
examples/expect [expect]
]
#end here.

The -s option can also take a filename wildcard as an argument

|pat.rb -s examples/xml.pat

Use -x debug and -v 10 to print out details of matching.

Using -x time will allow logging of timeduration of tests

Using -x debug,xchars prints the string representation of unprintable characters
like \r \n etc.
